# What?

It separates your system into **core business logic** and **external world**. Hexagonal Architecture isnвҖҷt about folders. ItвҖҷs about **protecting your business logic from chaos**.

рҹ‘ү Your app should work **without NestJS, DB, or APIs**

Think:

* **Domain** вҶ’ pure logic (no framework)
* **Application** вҶ’ use cases
* **Ports** вҶ’ interfaces
* **Adapters** вҶ’ implementations
* **Infrastructure** вҶ’ framework glue

# Explained (Step-by-Step)

> Request hits controller вҶ’ controller calls use case вҶ’ use case executes business logic via domain вҶ’ interacts with repository interfaces вҶ’ infrastructure provides implementations (DB/API).
> This keeps business logic independent from frameworks.

## 1. Request вҶ’ Controller

рҹ‘ү A user (or frontend) sends a request.    
рҹ‘ү Controller in NestJS receives it

Example:

```http
POST /bookings
```

```js
@Post()
create(@Body() body: CreateBookingRequest) {
  return this.useCase.execute(body);
}
```

**What controller does:** Validate input, Call use case, Return response, No business logic here.

## 2. Controller вҶ’ Use Case (Application layer)

рҹ‘ү This is the **brain of the flow**.    
рҹ‘ү Think: вҖңWhat should happen when user books?вҖқ

```ts
execute(dto: CreateBookingDto) {
  // orchestrate everything
}
```

**What use case does:** Coordinates steps, Calls domain logic, Calls repository (interface) 

## 3. Use Case вҶ’ Domain (Business rules)

рҹ‘ү Domain decides **what is valid or not**.    
рҹ‘ү Think: вҖңIs this booking valid?вҖқ

```ts
if (booking.conflictsWith(existing)) {
  throw new Error('Conflict');
}
```

**What domain does:** Contains rules, Validates logic,  No DB / no framework

## 4. Use Case вҶ’ Repository (Interface / Port)

рҹ‘ү Use case doesnвҖҷt know DB вҶ’ it uses an **interface**.    
рҹ‘ү Think: вҖңI need data, but I donвҖҷt care howвҖқ

```ts
this.bookingRepo.save(booking);
```

**Important:** This is just a contract. No implementation here

## 5. Infrastructure (Implementation)

рҹ‘ү Real implementation lives here.    
рҹ‘ү Think: вҖңHow do we actually do it?вҖқ

```ts
@Injectable()
export class BookingRepositoryImpl implements BookingRepository {
  async save(booking: Booking) {
    return this.prisma.booking.create(...);
  }
}
```

**What infrastructure does:** DB queries (Prisma), API calls (payment, email), External systems
