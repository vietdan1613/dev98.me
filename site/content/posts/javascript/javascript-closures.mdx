## Scope

### What is Scope?

**Scope** defines **where a variable can be accessed** in your code.

JavaScript has:

* **Global scope** â€“ variables available everywhere
* **Function scope** â€“ variables inside a function
* **Block scope** â€“ variables inside `{}` when using `let` or `const`

```js

  let x = 10; // function scope
```

### Lexical Scope


## closure

A **closure** happens when a function **remembers variables from its outer scope**, even after that outer function has finished running.

```js
function createCounter() {
  let count = 0;
  return function () {
    return ++count;
  };
}

const counter = createCounter();
counter(); // 1
counter(); // 2
```

### Why Closures Matter

Closures are powerful because they enable:

* **Data encapsulation** (private variables)
* **State preservation**
* **Callbacks & event handlers**
* **React hooks and components**

Example of data privacy:

```js
function createUser() {
  let password = "secret";
  return input => input === password;
}
```
